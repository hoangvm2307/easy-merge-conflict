When two people edit the same scene or prefab, Git hands you a wall of fileID: 1001 and guid: a1b2c3… with no hint of which GameObject is in conflict. How it works

It reads the clean versions, not the markers

Instead of parsing the corrupted <<<<<<< markers in your working copy, it asks Git for the three clean versions of the file — base, ours, theirs — and diffs them structurally, by object, property, list element and field.

1 · Hit a conflict

Merge, rebase, cherry-pick or revert that touches a Unity YAML asset or a .meta file.

2 · Open the window

Tools ▸ Easy Merge Conflict lists every conflicted asset with readable names and a per-file count.

3 · Decide & stage

Pick Ours/Theirs per conflict, then Apply & Stage. Finish the merge (commit) in your Git client.

Honest by design

What it does — and doesn't

Does

  • Structural 3-way merge of scenes, prefabs and other YAML assets
  • Per-property, per-list-element and per-field resolution
  • Merges .meta importer settings + repoints GUID-clash references project-wide
  • Resolves & stages the asset, then you finish the merge in your Git client
  • Editor-only — ships nothing into your build

Doesn't (by design)

  • Doesn't run commit / --continue / --abort — you finish in your Git client
  • Very deeply-nested both-sides edits surface as one whole-block choice (never a silent drop)
  • Doesn't merge binary-serialized assets — use Force Text serialization
  • Not a general Git workflow tool or a replacement for your VCS
Good to know

Requirements & FAQ

Unity 2022.3 or newer · Git on your PATH (or point the tool at a git binary) · Assets serialized as Force Text (Unity's default for source control).

How is this different from Unity's built-in Smart Merge (UnityYAMLMerge)?
Smart Merge is a command-line merge driver you configure in Git, and it still resolves to raw YAML you can't easily read or review. Easy Merge Conflict is a visual Editor window: it shows conflicts by GameObject and property with names resolved, lets you decide each one, reconstructs the resulting Hierarchy, and warns about broken references before you write.
Does it change my Git workflow?
No. It uses the Git you already have and works with any workflow — GitHub, GitLab, Plastic-over-Git, Fork, Sourcetree, command line. It resolves and stages the conflicted asset; you commit / continue the merge however you normally do.
What happens with a .meta GUID conflict?
When each side gave an asset a different GUID, the tool shows both GUIDs with how many references each has. You pick the side to keep, and every reference to the other GUID is repointed across the project so nothing breaks. Those cross-file rewrites are written but left unstaged for you to review; only the .meta itself is staged.
Is anything included in my game build?
No. It's an Editor-only tool — nothing ships into your runtime build.
Can it corrupt my files?
It backs up the working-copy file before every Apply, validates that the merged asset re-imports cleanly, and offers one-click Restore. Multi-file GUID rewrites are backed up as a set and rolled back automatically if any write fails.
